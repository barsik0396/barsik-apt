# Контрибуция в barsik-apt

# Как начать
Для начала, создаём свой форк barsik-apt.

Дальше клонируем его:
```bash
gh repo clone user/repo
```
Или если у вас нет gh cli:
```bash
git clone https://github.com/user/repo.git
```

# Как создать свой пакет
Для создания своего пакета в barsik-apt, надо `.deb`, форк, скрипты...
Давайте разберём всё отдельно.

Для публикации на barsik-apt, да и на любые другие apt-сервера надо сам файл пакета.

Если у вас ещё нет deb пакета, можете загуглить как собрать deb - там всё поймёте.

Дальше просто копируем `.deb` в папку с форком.

После копирования, запускаем:
```bash
./add_package.sh *.deb --channel community
```
Это добавит .deb в pool/community и обновит репозиторий.

Отлично, теперь мы добавили пакет в barsik-apt!

# Правила коммитов
Для разных вещей разные коммиты.

Для pool/ и dists/: `update repository`

Для других свои правила:
1. По возможности коммиты только на английском
2. Только маленькими буквами
